# Niche Taxonomy — Standardized Category & Subcategory Tree

> This file provides a standardized classification tree so all agents (Discovery, Saturation, Pricing,
> Review, etc.) classify products consistently into the same categories and subcategories.
> The Discovery Agent maps products into this tree during identification; downstream agents use the
> mapped category to retrieve relevant KB content and apply category-specific heuristics.
>
> Categories at depth 1 (broad) and depth 2 (specific) are defined here. Agents should map products
> to the most specific depth-2 subcategory available. If no depth-2 match exists, use depth-1.

## Fallback Classification Rule

If a product cannot be confidently mapped to any depth-2 category (confidence < 60%), use only the depth-1 category and flag the product as "broad classification — low category specificity" in the agent's reasoning. This prevents false precision where an agent guesses a subcategory it isn't confident about, which would then propagate incorrect category-specific heuristics to downstream agents.

Per Architecture Section 4.2's standard: it is better to be honestly broad than falsely precise.
